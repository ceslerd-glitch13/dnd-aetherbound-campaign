# Creature Repository

Use this folder as the single entry point for Elyndria's official-creature catalog.

## Start here

- [`roll_tables/Roll_Table_Index.md`](roll_tables/Roll_Table_Index.md) — mandatory first stop whenever a random encounter roll is requested.
- [`roll_tables/Roll_Table_Selection_Guide.md`](roll_tables/Roll_Table_Selection_Guide.md) — chooses an appropriate d6 table for a region or situation.
- [catalog index](Creature_Catalog_Index.md) — alphabetical directory, category map, authority rules, and deferred decisions.
- [`lookups/Creature_Regional_Lookup.md`](lookups/Creature_Regional_Lookup.md) — region, habitat, and major-city search.
- [`lookups/Creature_Encounter_Lookup.md`](lookups/Creature_Encounter_Lookup.md) — CR bands and gate-city suitability.
- [`lookups/Creature_Dead_Zone_and_Source_Lookup.md`](lookups/Creature_Dead_Zone_and_Source_Lookup.md) — physical viability, magical dependency, DM adjudication, and source baseline.
- `data/creatures.json` — structured catalog record for AI and scripts.
- `data/creatures.csv` — spreadsheet-friendly mirror.

## Repository boundaries

The creature catalog helps select and place official creatures. It does not replace:

- `../canon/DnD_Campaign_World_Bible.md` for world mechanics and dead zones;
- `../canon/World_Geography.md` for spatial canon;
- an owned official source for a full published stat block;
- `../combat/Enemy_Encounters_Stat_Blocks.md` for custom campaign statistics.

Category pages and roll tables are supporting views. The main creature authority and navigation rules are in `Creature_Catalog_Index.md`; structured creature records are synchronized in `data/creatures.json`.

When a roll is requested, use the roll-table section first, roll the result, and only then open the linked creature entries for additional information.
